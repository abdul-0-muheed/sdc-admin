## Database Fix (Supabase SQL):
The error "new row violates row-level security policy for table 'site_settings'" occurs because there is no policy allowing `INSERT` or `UPDATE` (upsert) on the `site_settings` table. 

Run the following SQL in your Supabase SQL Editor to fix it:

```sql
-- 1. Ensure RLS is enabled
ALTER TABLE public.site_settings ENABLE ROW LEVEL SECURITY;

-- 2. Allow everyone to read settings (for the public website)
CREATE POLICY "Allow public read access" ON public.site_settings
FOR SELECT TO public USING (true);

-- 3. Allow all operations for authenticated users (for the admin panel)
-- Note: If you are not using Supabase Auth, change 'authenticated' to 'anon'
CREATE POLICY "Allow admin full access" ON public.site_settings
FOR ALL TO authenticated USING (true) WITH CHECK (true);
```
